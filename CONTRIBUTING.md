# Contributing

Thank you for contributing to JobTracks AI.

This project follows a simple workflow to keep development organized and maintain code quality.

## Branch Strategy

Do not commit directly to the `main` branch.

Create a new branch for each feature or fix.

Examples:

feature/auth-api  
feature/application-crud  
fix/login-validation  
docs/update-readme  

## Commit Message Format

Use clear commit messages describing the change.

Examples:

feat: add job application model  
fix: resolve login validation issue  
docs: update architecture documentation  
chore: update dependencies  

## Development Workflow

1. Create a branch
2. Make your changes
3. Commit with a clear message
4. Push the branch
5. Open a Pull Request

## Code Style

- Use Prettier for formatting
- Follow ESLint rules for JavaScript/TypeScript
- Write clear and readable code

## Documentation

If you add major features, update the documentation in the `docs/` folder.
